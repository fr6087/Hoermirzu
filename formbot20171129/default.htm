LuisBot

LuisBot implements diffrent features of Microsoft Bot Framework's FormFlow: it shows how to generate a form dialoge from the file "JsonDummyForBot.json" and how to connect multiple dialogs by options. Also there are functions for gernerating localized dialogs.

You can run this bot in the Cortana canvas, if you set your device's language and region's settings to USA and are within the group of users to which the bot was published. Type "ask formbot" in the Cortana Canvas to give it a try. There is also the possibility to connect via DirectLine to this bot. This is shown in one of this project's sibling projects, WCFService1.

creation story of this bot

for reproduction of this bot as a cortana skill

  1. register to Azure Portal with valid credit card number.
  2. go to create Application/AI and cognitive/Web App Bot. Follow the steps to create a bot and be careful to select F0 as pricing tier. If the option is not available, change to F0 after creation.
  3. publish bot from Azure to Cortana dashboard by Cortana channel
  4. register as a Cortana developer (cost as an individual: 19 USD)
  5. Enable Debug mode in the Cortana Developer Center. and set your MSA email address as a developer so the skill gets visible to you.
  6. check that you are logged in with the same MSA account used on azure/cortana dashboard in the cortana settings
  7. call the skill by it's invocation name with "tell [invocation name] "
This bot is available for other channels as well. If you do not want to connect to Cortana, you may use Skype, Facebook messenger or some other channel.

How to debug the bot

You can debug this bot locally with the help of the Bot Framework Emulator, if you are online and not behind a proxy. For changes on the LUIS language model please Visit the LUIS website and login with the Microsoft Account that was used to host this bot in Azure.

  • add new languages to the bot. E.g. for Polish create a ressource file Resource1.pl.resx in Resource folder. Right click select that file, set build action to embedded resource and userdefined tool to ResXFileCodeGeneraror. type Windows key and search for visual studio command prompt. Navigate to the Resource folder and execute the commands below. Afterwards, ensure that the ResourceManager property in Resource1.cs is naming the namespace and the filename as first argument.
